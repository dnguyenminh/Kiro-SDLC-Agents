## 1. Introduction

### 1.1 Purpose

This FSD specifies the functional behavior of UX improvements for KB Web Viewer, covering 7 features: Onboarding Tour, Contextual Tooltips, Smart Empty States, Actionable Recommendations, Graph Auto-Analysis, User Guide Integration, and UI Redesign.

### 1.2 Scope

All changes are frontend-only (HTML/CSS/JS) with minimal new API endpoints. The existing Python HTTP server serves pages via string templates; new JS modules will be loaded as separate files from `shared/viewer/`.

### 1.3 Definitions & Acronyms

| Term | Definition |
|------|------------|
| Tour | Step-by-step onboarding overlay |
| Tooltip | Contextual help popup on hover/click |
| Empty State | UI shown when section has no data |
| Insight | Auto-detected pattern in graph data |
| Help Panel | Slide-in panel with guide content |

## 2. System Overview

### 2.1 System Context

The KB Web Viewer is a single-page-per-route web application served by a Python HTTP server on port 3201. Each page is a complete HTML document generated by Python string templates. JavaScript handles interactivity client-side.

```
┌─────────────────────────────────────────────────────────┐
│ Browser (localhost:3201)                                 │
│ ┌─────────┐ ┌──────────┐ ┌──────┐ ┌───────┐ ┌────────┐│
│ │  Graph  │ │Dashboard │ │ Tags │ │Quality│ │Analytics││
│ │(4 tabs) │ │          │ │      │ │       │ │        ││
│ └────┬────┘ └────┬─────┘ └──┬───┘ └───┬───┘ └───┬────┘│
│      │           │           │         │         │      │
│ ┌────▼───────────▼───────────▼─────────▼─────────▼────┐│
│ │              UX Enhancement Layer (NEW)               ││
│ │  tour.js | tooltips.js | empty-states.js |           ││
│ │  recommendations.js | graph-analysis.js |            ││
│ │  help-panel.js | ui-tokens.css                       ││
│ └──────────────────────────┬───────────────────────────┘│
└────────────────────────────┼────────────────────────────┘
                             │ fetch() API calls
┌────────────────────────────▼────────────────────────────┐
│ Python HTTP Server (BaseHTTPRequestHandler)              │
│ ├── /api/kb/dashboard, /api/kb/tags, /api/kb/quality    │

└─────────────────────────────────────────────────────────┘
```

### 2.2 Architecture Constraints

- **No build tools** — all JS/CSS served as-is (no webpack, no bundler)
- **No frameworks** — vanilla JS with ES6 modules (`<script type="module">`)
- **Max 200 lines per file** — split into focused modules
- **Dark theme only** — CSS custom properties for theming tokens
- **Server-rendered HTML** — Python generates full HTML, JS enhances client-side

## 3. Functional Requirements

### 3.1 Feature: Onboarding Tour (KSA-94)

**Source:** BRD Story 1

#### 3.1.1 Description

A step-by-step guided tour that highlights key UI elements for first-time users. The tour uses a spotlight overlay pattern — dimming the background while illuminating the target element with a tooltip explanation.

#### 3.1.2 Use Case

**Use Case ID:** UC-01
**Actor:** First-time User
**Preconditions:** User has never visited the viewer (no `kb-viewer-tour-completed` in localStorage)
**Postconditions:** Tour flag saved in localStorage, user understands basic navigation

**Main Flow:**

| Step | Actor | System | Description |
|------|-------|--------|-------------|
| 1 | Opens viewer URL | Detects first visit | Check localStorage for tour flag |
| 2 | | Shows tour overlay | Dim background, spotlight first element |
| 3 | Clicks "Next" | Advances to next step | Move spotlight to next element |
| 4 | | Shows step N/total | Update progress indicator |
| 5 | Clicks "Next" on last step | Completes tour | Save flag, remove overlay |

**Alternative Flows:**

| ID | Condition | Steps |
|----|-----------|-------|
| AF-1 | User clicks "Skip Tour" | Save flag as "skipped", remove overlay immediately |
| AF-2 | User clicks "Previous" | Go back one step, update spotlight |
| AF-3 | User navigates to different page during tour | Pause tour, resume when returning |

**Exception Flows:**

| ID | Condition | Steps |
|----|-----------|-------|
| EF-1 | Target element not found in DOM | Skip that step, advance to next |
| EF-2 | localStorage unavailable | Show tour every time (graceful degradation) |

#### 3.1.3 Business Rules

| Rule ID | Rule | Source |
|---------|------|--------|
| BR-01 | Tour shows only once per browser (localStorage-based) | KSA-94 |
| BR-02 | Tour can be restarted from Help menu | KSA-94 |
| BR-03 | Tour adapts steps based on current page | KSA-94 |
| BR-04 | Tour does not block underlying page functionality when dismissed | KSA-94 |

#### 3.1.4 Data Specifications

**Tour Configuration (JSON):**

| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| steps | Array | Y | min 1 item | Array of tour step objects |
| steps[].id | String | Y | unique | Step identifier |
| steps[].target | String | Y | CSS selector | Element to highlight |
| steps[].title | String | Y | max 50 chars | Step title |
| steps[].content | String | Y | max 200 chars | Step description |
| steps[].position | Enum | Y | top/bottom/left/right | Tooltip position relative to target |
| steps[].page | String | N | valid page path | Page this step belongs to |

### 3.2 Feature: Contextual Tooltips (KSA-95)

**Source:** BRD Story 2

#### 3.2.1 Description

Small ? icons placed next to UI components. On hover (desktop) or tap (touch), a tooltip popup appears with a concise explanation of what the component does and how to use it.

#### 3.2.2 Use Case

**Use Case ID:** UC-02
**Actor:** Any User
**Preconditions:** Page is loaded, tooltip targets exist in DOM
**Postconditions:** User understands the component's purpose

**Main Flow:**

| Step | Actor | System | Description |
|------|-------|--------|-------------|
| 1 | Hovers over ? icon | Shows tooltip | Tooltip appears with 200ms delay |
| 2 | Reads tooltip content | Maintains tooltip | Tooltip stays while hovering |
| 3 | Moves mouse away | Hides tooltip | Tooltip fades out in 100ms |

**Alternative Flows:**

| ID | Condition | Steps |
|----|-----------|-------|
| AF-1 | Touch device — user taps ? | Show tooltip, dismiss on tap elsewhere |
| AF-2 | Tooltip would overflow viewport | Reposition to opposite side |

#### 3.2.3 Business Rules

| Rule ID | Rule | Source |
|---------|------|--------|
| BR-05 | Tooltips appear within 200ms of hover | KSA-95 |
| BR-06 | Tooltips disappear within 100ms of mouse-out | KSA-95 |
| BR-07 | Only one tooltip visible at a time | KSA-95 |
| BR-08 | Tooltip content max 3 sentences | KSA-95 |

## 6. Processing Logic

### 6.1 Graph Analysis Algorithm (Client-Side)

**Trigger:** Graph page loads with data, or user clicks "Analyze"
**Input:** nodes[] and edges[] from graph API
**Output:** insights[] with affected node IDs

**Processing Steps:**

| Step | Description | Error Handling |
|------|-------------|----------------|
| 1 | Build adjacency list from edges | Skip invalid edges |
| 2 | Find orphans: nodes with degree 0 | Always succeeds |
| 3 | Find hubs: nodes with degree > 10 | Always succeeds |
| 4 | Run BFS to find connected components | Timeout after 2s |
| 5 | Identify large clusters (component size > 20) | Always succeeds |
| 6 | Check node freshness (updated_at > 180 days ago) | Skip if no date |
| 7 | Compile insights array | Return partial results on timeout |

### 6.2 Recommendation Engine (Server-Side)

**Trigger:** GET /api/kb/recommendations
**Input:** All entries from KB
**Output:** Prioritized recommendation list

**Processing Steps:**

| Step | Description | Error Handling |
|------|-------------|----------------|
| 1 | Query entries with quality_score < 40 | Return empty on DB error |
| 2 | Query entries with 0 tags | Return empty on DB error |
| 3 | Query entries not reviewed in 90+ days | Return empty on DB error |
| 4 | Query entries with 0 relationships | Return empty on DB error |
| 5 | Assign severity based on issue type | Default to "medium" |
| 6 | Sort by severity, limit to 10 | Always succeeds |
| 7 | Return recommendation objects with action definitions | Always succeeds |

## 9. Error Handling (User-Facing)

### 9.1 Error Scenarios

| Scenario | Severity | User Message | Expected Behavior |
|----------|----------|-------------|-------------------|
