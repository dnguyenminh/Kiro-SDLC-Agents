### PBT-08: Permission Rule Validation Properties

| Field | Value |
|-------|-------|
| **ID** | PBT-08 |
| **Priority** | High |
| **Type** | Property-Based Test |
| **Requirement** | BR-40, BRD Permission Rules Specification |
| **Preconditions** | PermissionRuleValidator and RuleDefinitionRegistry available |

**Properties:**
1. Boolean rule with `true`/`false` → VALID
2. Boolean rule with non-boolean value (string, number) → INVALID
3. Number rule within min–max range → VALID
4. Number rule below min → INVALID
5. Number rule above max → INVALID
6. Number rule with float/decimal → INVALID (must be integer)
7. Enum[] rule with values all in allowedValues list → VALID
8. Enum[] rule with value NOT in allowedValues → INVALID
9. Enum[] rule with empty array → INVALID (at least one required)
10. Unknown rule name (not in definitions for permission) → INVALID
11. LOCKED rule submitted with non-default value → IGNORED (server uses default)

**Test Data:** 
- Generate random roleData payloads for each of 14 permissions (500 iterations)
- Mix valid/invalid values per rule type
- Test boundary conditions: `maxEntries=0`, `maxEntries=100001`, `maxNodes=49`, `maxNodes=5001`
