## 2. Database Layer

### 2.1 Database: SQLite Embedded (`admin.db`)

Sử dụng SQLite embedded trong cùng process — không cần external database server.

**Ưu điểm so với Neo4j:**
- Zero configuration, zero network overhead
- ACID transactions built-in
- Single file backup/restore
- Spatial indexing qua compound indexes trên x, y, z
- Query < 100ms cho 72k+ nodes

### 2.2 Schema

```sql
-- Node table: mỗi KB entry = 1 node với 3D position
CREATE TABLE graph_nodes (
  entry_id TEXT PRIMARY KEY,
  label TEXT,
  type TEXT,
  tier TEXT,
  x REAL,
  y REAL,
  z REAL,
  level INTEGER,
  cluster_id TEXT
);

-- Edge table: relationships giữa nodes
CREATE TABLE graph_edges (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source TEXT NOT NULL,
  target TEXT NOT NULL,
  weight REAL DEFAULT 1.0,
  rel_type TEXT,
  UNIQUE(source, target),
  FOREIGN KEY (source) REFERENCES graph_nodes(entry_id),
  FOREIGN KEY (target) REFERENCES graph_nodes(entry_id)
);

-- Performance indexes
CREATE INDEX idx_nodes_x ON graph_nodes(x);
CREATE INDEX idx_nodes_y ON graph_nodes(y);
CREATE INDEX idx_nodes_z ON graph_nodes(z);
CREATE INDEX idx_nodes_level ON graph_nodes(level);
CREATE INDEX idx_nodes_cluster ON graph_nodes(cluster_id);
CREATE INDEX idx_edges_source ON graph_edges(source);
CREATE INDEX idx_edges_target ON graph_edges(target);
CREATE INDEX idx_edges_pair ON graph_edges(source, target);
```

### 2.3 Layout Algorithm: Cluster-Based Disk Scatter

Mỗi `type` được gán một cluster center riêng biệt (cách nhau radius 600 units). Nodes trong cùng cluster được scatter theo disk pattern (không phải sphere) để tạo hình dạng tự nhiên.

```
Cluster Centers (ví dụ):
  "code"       → (0, 0, 0)
  "document"   → (600, 0, 0)
  "config"     → (-600, 0, 0)
  "test"       → (0, 0, 600)
  "dependency" → (0, 0, -600)
  ...

Node position = cluster_center + random_disk_offset(radius=200)
```

### 2.4 Edge Generation: Bucket-Based k-NN

Edges được tạo dựa trên spatial proximity thay vì type relationships:
- Chia không gian 3D thành buckets
- Mỗi node kết nối với k nearest neighbors trong cùng bucket
- Weight = inverse distance (gần hơn = weight cao hơn)

## 3. Rendering Layer — Hybrid Three.js Renderer

### 3.1 File: `kb-graph-renderer.js`

Renderer sử dụng 3 LOD (Level of Detail) modes dựa trên camera distance:

| Mode | Camera Distance | Rendering Method | Nodes Shown | Draw Calls |
|------|----------------|-----------------|-------------|------------|
| **FAR** | > 800 units | `THREE.Points` | ALL nodes | 1 |
| **MID** | 300–800 units | `InstancedMesh` + `Points` | 2000 nearest (instanced) + rest (points) | 2 |
| **CLOSE** | < 300 units | Individual `Spheres` + Labels + `Points` | 500 nearest (spheres+labels) + rest (points) | ~502 |

### 3.2 FAR Mode (distance > 800)

```javascript
// Tất cả nodes render bằng 1 Points object
const geometry = new THREE.BufferGeometry();
geometry.setAttribute('position', new THREE.Float32BufferAttribute(positions, 3));
geometry.setAttribute('color', new THREE.Float32BufferAttribute(colors, 3));

const material = new THREE.PointsMaterial({
  size: 3,
  vertexColors: true,
  sizeAttenuation: true,
  transparent: true,
  opacity: 0.8
});

const points = new THREE.Points(geometry, material);
scene.add(points);
```

**Đặc điểm:**
- 1 draw call duy nhất — supports millions of nodes
- Mỗi node = 1 glowing colored dot
- Color mapped theo `type` (code=cyan, doc=green, config=yellow, etc.)
- Edges HIDDEN — tránh wireframe effect

### 3.3 MID Mode (distance 300–800)

```javascript
// Top 2000 nearest nodes → InstancedMesh
const sphereGeo = new THREE.SphereGeometry(2, 8, 8);
const instancedMesh = new THREE.InstancedMesh(sphereGeo, material, 2000);

// Set matrix for each instance
for (let i = 0; i < nearestNodes.length; i++) {
  const matrix = new THREE.Matrix4();
  matrix.setPosition(node.x, node.y, node.z);
  instancedMesh.setMatrixAt(i, matrix);
  instancedMesh.setColorAt(i, getColorForType(node.type));
}

// Remaining nodes → Points (same as FAR)
```

**Đặc điểm:**
- InstancedMesh cho nearby nodes (chi tiết hơn, vẫn 1 draw call cho batch)
- Points cho distant nodes
- Edges SHOWN cho instanced nodes (LineSegments)

### 3.4 CLOSE Mode (distance < 300)

```javascript
// Top 500 nearest nodes → Individual spheres + labels
for (const node of nearest500) {
  const sphere = new THREE.Mesh(
    new THREE.SphereGeometry(3, 16, 16),
    new THREE.MeshPhongMaterial({ color: getColorForType(node.type) })
  );
  sphere.position.set(node.x, node.y, node.z);
  sphere.userData = { entryId: node.entry_id, label: node.label };
  scene.add(sphere);

  // CSS2D Label
  const label = createLabel(node.label);
  sphere.add(label);
}
```

**Đặc điểm:**
- Individual spheres với Phong shading
- CSS2D text labels hiển thị node name
- Edges shown với opacity gradient
- Full raycasting support cho click interaction

### 3.5 LOD Transition

```javascript
// Throttled 2 seconds — tránh flickering khi scroll
let lastLodCheck = 0;
const LOD_THROTTLE = 2000; // ms

function onCameraMove() {
  const now = Date.now();
  if (now - lastLodCheck < LOD_THROTTLE) return;
  lastLodCheck = now;

  const dist = camera.position.distanceTo(focusPoint);
  const newMode = dist > 800 ? 'FAR' : dist > 300 ? 'MID' : 'CLOSE';
  
  if (newMode !== currentMode) {
    transitionToMode(newMode);
  }
}
```

### 3.6 Node Interaction (Raycasting)

```javascript
// Raycasting hỗ trợ cả 3 LOD modes
const raycaster = new THREE.Raycaster();
raycaster.params.Points.threshold = 5; // FAR mode click radius

function onClick(event) {
  raycaster.setFromCamera(mouse, camera);
  
  // Check Points (FAR/MID)
  const pointIntersects = raycaster.intersectObject(pointsObject);
  if (pointIntersects.length > 0) {
    const index = pointIntersects[0].index;
    showNodeDetail(nodes[index]);
    return;
  }
  
  // Check InstancedMesh (MID)
  const instanceIntersects = raycaster.intersectObject(instancedMesh);
  if (instanceIntersects.length > 0) {
    const instanceId = instanceIntersects[0].instanceId;
    showNodeDetail(nearestNodes[instanceId]);
    return;
  }
  
  // Check individual spheres (CLOSE)
  const sphereIntersects = raycaster.intersectObjects(sphereGroup.children);
  if (sphereIntersects.length > 0) {
    showNodeDetail(sphereIntersects[0].object.userData);
  }
}
```

**Node Detail Panel:** Click vào node hiển thị panel với markdown rendering (label, type, tier, content preview).

## 4. API Endpoints

### 4.1 `GET /api/admin/kb/graph/positions`

Trả về ALL node positions, filtered theo user's `allowedTiers`.

**Request:**
```
GET /api/admin/kb/graph/positions
Authorization: Bearer <token>
```

**Response:**
```json
{
  "nodes": [
    {
      "entry_id": "abc-123",
      "label": "SqliteGraphService.ts",
      "type": "code",
      "tier": "internal",
      "x": 142.5,
      "y": -23.1,
      "z": 601.8,
      "level": 1,
      "cluster_id": "code"
    }
  ],
  "totalCount": 72000,
  "filteredCount": 65000
}
```

**RBAC:** Chỉ trả về nodes mà user có quyền KB_READ trên tier đó.

### 4.2 `GET /api/admin/kb/graph/spatial`

Spatial query với LOD levels — trả về nodes gần camera position.

**Request:**
```
GET /api/admin/kb/graph/spatial?x=100&y=0&z=200&zoom=500
Authorization: Bearer <token>
```

**Query Parameters:**

| Param | Type | Description |
|-------|------|-------------|
| x | number | Camera focal point X |
| y | number | Camera focal point Y |
| z | number | Camera focal point Z |
| zoom | number | Camera distance (dùng để chọn LOD) |

**Response:**
```json
{
  "mode": "MID",
  "detailed": [...],
  "summary": {
    "totalNodes": 72000,
    "pointPositions": [...]
  },
  "edges": [
    { "source": "abc-123", "target": "def-456", "weight": 0.85 }
  ]
}
```

### 4.3 `POST /api/admin/kb/graph/sync`

Rebuild graph từ KB entries. Tính toán lại layout positions cho tất cả nodes.

**Request:**
```
POST /api/admin/kb/graph/sync
Authorization: Bearer <token>
```

**Response:**
```json
{
  "success": true,
  "nodesCreated": 72000,
  "edgesCreated": 215000,
  "durationMs": 3400
}
```

**Behavior:**
- Truncate `graph_nodes` + `graph_edges`
- Read ALL KB entries
- Compute cluster-based disk scatter layout
- Generate edges via bucket-based k-NN
- Insert all nodes + edges trong 1 transaction

## 5. RBAC Integration

### 5.1 Filtering Logic

```typescript
// Positions filtered by user's KB_READ.allowedTiers
function getFilteredPositions(userId: string) {
  const user = getUserRBAC(userId);
  const allowedTiers = user.permissions.KB_READ.allowedTiers; // e.g. ["public", "internal"]
  
  return db.prepare(`
    SELECT * FROM graph_nodes 
    WHERE tier IN (${allowedTiers.map(() => '?').join(',')})
  `).all(...allowedTiers);
}
```

### 5.2 Real-time Update

Khi RBAC thay đổi (admin cập nhật user permissions), renderer auto-refresh:

```javascript
// Frontend listens for RBAC changes
window.addEventListener('rbac-changed', (event) => {
  // Re-fetch positions with new permissions
  fetchPositions().then(data => {
    renderer.updateNodes(data.nodes);
  });
});

// Backend dispatches CustomEvent khi RBAC update
function onRBACUpdate(userId) {
  broadcastToUser(userId, { type: 'rbac-changed' });
}
```

## 6. Progressive Loading & Performance

### 6.1 Loading Strategy

1. **Initial load:** Fetch `/positions` → render FAR mode (Points only)
2. **Camera move:** Throttled 2s check → nếu mode thay đổi, fetch `/spatial`
3. **Zoom in:** MID → fetch top 2000 nearest + edges
4. **Zoom close:** CLOSE → fetch top 500 nearest + labels + edges

### 6.2 Performance Benchmarks

| Metric | Value | Condition |
|--------|-------|-----------|
| Initial render | < 200ms | 72k nodes, FAR mode |
| LOD transition | < 100ms | Mode switch |
| Spatial query | < 100ms | SQLite with indexes |
| Memory (FAR) | ~50MB | 72k nodes as Points |
| Memory (CLOSE) | ~150MB | 500 spheres + labels + 72k points |
| Target FPS | 60fps | Consistent across all modes |
| Scalability | 1M+ nodes | Points rendering = 1 draw call |

### 6.3 Optimization Techniques

- **Float32Array** cho position/color buffers (không dùng objects)
- **Frustum culling** tự động bởi Three.js
- **InstancedMesh** thay vì individual meshes ở MID mode
- **Throttled LOD check** (2s) tránh rapid re-renders
- **SQLite spatial indexes** cho range queries
- **Batch INSERT** trong transaction khi sync

## 10. Auto-Sync Behavior

### 10.1 Startup Sync

```typescript
// KBGraphModule.ts
async onStartup() {
  const count = db.prepare('SELECT COUNT(*) as cnt FROM graph_nodes').get();
  if (count.cnt === 0) {
    console.log('[KB Graph] No nodes found, triggering initial sync...');
    await this.graphService.syncFromKB();
    console.log('[KB Graph] Initial sync complete');
  }
}
```

### 10.2 Manual Sync

Admin có thể trigger manual sync qua:
- API: `POST /api/admin/kb/graph/sync`
- UI: Button "Rebuild Graph" trong KB Graph page

### 10.3 Incremental vs Full Rebuild

Hiện tại sử dụng **full rebuild** (truncate + re-insert) vì:
- SQLite transaction đủ nhanh (< 5s cho 72k nodes)
- Layout cần recompute khi thêm/xóa nodes
- Simpler logic, ít edge cases

## 11. Kết Luận

Implementation sử dụng approach "zero external dependencies" — toàn bộ graph storage + visualization chạy embedded trong existing backend process. SQLite cho persistence, Three.js cho rendering, cluster-based layout cho visual organization.

**Key metrics đạt được:**
- ✅ 72k nodes @ 60fps
- ✅ Spatial queries < 100ms  
- ✅ Zero external dependencies (no Neo4j, no graph DB)
- ✅ RBAC-aware filtering
- ✅ Progressive loading (FAR → MID → CLOSE)
- ✅ Click interaction across all LOD modes
- ✅ Designed for 1M+ nodes (Points rendering)
